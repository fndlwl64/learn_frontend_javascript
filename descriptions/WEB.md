## HTML
### Web
#### World Wide Web
- 인터넷 상에서 문서(Document)를 하이퍼텍스트 방식으로 연결해 제공하는 공간
#### Web Page
- HTML 언어를 사용하여 작성된 형태로 제공된 문서
#### Web Site
- 웹 페이지 중 서로 관련된 내용으로 작성된 웹 페이지들의 집합

---

### Web Browser
- 웹 서버에서 HTTP를 통해 양방향 통신
- HTML 문서 및 자원(Resource) 등의 컨텐츠 제공
- GUI 기반의 소프트웨 프로그램

---

### Web의 동작원리
#### Web Browser의 동작 방식
- 요청과 응답(request, response)
    - 사용자가 선택한 자원을 서버에 요청하고 서버의 응답을 브라우저에 출력(Rendering)하는 방식으로 동작
    - > Tip - URI(Uniforme Resource Identifier) 형식으로 요청 주소 표시

#### Web Browser의 동작 과정
1. 사용자가 URL(Uniform Resource Locator) 주소 입력 
2. DNS 서버에서 사용자가 입력한 URL 주소 중 도메인 네임을 검색
    - > 해당하는 IP 주소를 찾아 사용자에게 URL과 함께 전달
3. HTTP 프로토콜을 사용해 URL과 IP 주소를 바탕으로 HTTP 요청 메시지를 생성
    - > TCP 프로토콜을 사용하여 인터넷을 거쳐 서버에 전송
    - > 서버에서 HTTP 프로토콜을 통해 HTTP 요청 메시지가 웹 페이지 URL 정보로 변환
    - > 서버가 변환된 URL 정보에 해당하는 데이터 검색
    - > HTTP 프로토콜을 통해 HTTP 응답 메시지 생성
    - > TCP 프로토콜을 이용해 메시지가 인터넷을 거쳐 클라이언트로 전송
4. HTTP 응답 메시지는 HTTP 프로토콜을 사용해 웹 페이지 데이터로 변환
5. 변환된 데이터는 웹 브라우저에서 출력

### Browser의 기본 구조와 특징
#### 기본 구조
- 사용자 인터페이스 > 브라우저 엔진 
    - 브라우저 엔진 > 자료저장소
    - 브라우저 엔진 > 렌더링 엔진
        - 렌더링 엔진 -> 통신
        - 렌더링 엔진 -> 자바스크립트 해석시
        - 렌더링 엔진 -> UI 벡엔드
#### 사용자 인터페이스(UI)
- 화면에 출력되어 사용자에게 직접 보여주는 영역
#### 브라우저 엔진
- 브라우저 엔진(Browser Engine)
  - 사용자 인터페이스와 렌더링 엔진 사이의 동작을 제어
  - 네트워크 요청, 보안 정책, 쿠키/캐시 관리 등 브라우저 전반을 조율
- 렌더링 엔진(Rendering Engine)
  - HTML, CSS, JavaScript 등을 파싱하여 DOM, CSSOM 트리를 생성
  - 레이아웃(Layout)과 페인팅(Painting) 과정을 거쳐 최종적으로 화면에 출력
- > Tip: 흔히 브라우저 엔진과 렌더링 엔진을 혼용하기도 하지만, 실제로는 역할이 구분됨
